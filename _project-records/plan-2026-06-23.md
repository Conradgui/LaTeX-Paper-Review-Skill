# Paper Review Skill Cleanup Plan

## Goal

Turn the provided paper-review bundle into a clean, publishable Codex skill inside this repository.

## Product Decision

Use `paper-review/` as the actual skill package directory because the repository folder name contains a space. This keeps the installable skill folder aligned with the skill name and avoids relying on the repository name.

## Scope

- Keep the skill focused on scientific and technical manuscript review.
- Preserve the useful bundled resources: workflow instructions, review rubric, and proofing scan script.
- Avoid extra README, installation guide, changelog, or process documentation inside the skill package.
- Keep project planning evidence outside the skill package in `_project-records/`.
- Keep generated review files, LaTeX build files, and Python cache files out of git.

## Validation Table

1. Create standard skill structure -> Verify `paper-review/SKILL.md`, `paper-review/agents/openai.yaml`, `paper-review/references/`, and `paper-review/scripts/` exist.
2. Replace template content -> Verify no TODO placeholders remain in the skill package.
3. Add proofing scan script -> Verify it runs on a small text sample and catches expected high-confidence issues.
4. Validate metadata -> Run skill-creator `quick_validate.py` against `paper-review/`.
5. Check repository cleanliness -> Run `find` and `git status --short` to confirm only expected files were added.

## Notes

- The skill should match the user's requested review language instead of hard-coding Chinese.
- PDF proofing scan depends on `pypdf`; text scanning should still work without it.
